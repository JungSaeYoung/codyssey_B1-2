## 1. OOM 실험

### 1-1. Before 조건 (장애 재현)

```bash
export MEMORY_LIMIT=256
export CPU_MAX_OCCUPY=95
export MULTI_THREAD_ENABLE=false
nohup ./agent-leak-app > $AGENT_LOG_DIR/agent-leak-app.log 2>&1 &
echo $! > /tmp/agent.pid
```

### 1-2. 관측 (10분 이상)

- `tail -f /var/log/agent-app/monitor.log` — MEM% 선형 증가 확인
- `tail -f $AGENT_LOG_DIR/agent-leak-app.log` — `MemoryGuard` / `SELF-TERMINATED` 라인 출현
- 종료 후 `ps -p $(cat /tmp/agent.pid)` → 사라짐 확인

### 1-3. After 조건 (조치 검증)

```bash
export MEMORY_LIMIT=512   # 256 → 512
# 재실행 후 동일 절차로 관측 → 생존시간 약 2배 증가 (10분 → 20분+)
```

## 2. CPU 과점유 실험

### 2-1. Before

```bash
export MEMORY_LIMIT=512
export CPU_MAX_OCCUPY=80
export MULTI_THREAD_ENABLE=false
nohup ./agent-leak-app > $AGENT_LOG_DIR/agent-leak-app.log 2>&1 &
```

### 2-2. 관측

- `top -bn1 -p <PID>` 5회 (30초 간격) → CPU% 비선형 상승 캡처

# 재실행 → 종료까지 약 4분 → 약 12분으로 연장 (여전히 종료됨)

## 3. Deadlock 실험

### 3-1. Before

```bash
export MEMORY_LIMIT=512
export CPU_MAX_OCCUPY=95
export MULTI_THREAD_ENABLE=true     # ★ 핵심
nohup ./agent-leak-app > $AGENT_LOG_DIR/agent-leak-app.log 2>&1 &
PID=$!
```

### 3-2. 관측 (2~5분 후 무응답 진입)

```bash
ps -ef | grep $PID                 # PID 존재 확인
ps -p $PID -o stat,etime,cmd       # Sl (sleeping + multi-thread)
top -H -bn1 -p $PID                # 스레드별 %CPU = 0.0
ps -L -p $PID -o lwp,pcpu,stat,wchan:25,cmd   # wchan=futex_wait_queue_me
tail -n 20 $AGENT_LOG_DIR/agent-leak-app.log  # WAITING/BLOCKED 라인에서 정지
curl --max-time 3 http://0.0.0.0:15034        # 타임아웃 — 외부 무응답 입증
```

### 3-3. After

```bash
export MULTI_THREAD_ENABLE=false
# 재실행 → 30분 이상 정상 가동, curl 응답 정상
```

## 4. 보너스 — 스케줄링 추론용 데이터 수집

위 실험 사이사이 **정상 가동 구간**(데드락 미발생, OOM 직전 충분히 떨어진 구간)에서:

```bash
export MULTI_THREAD_ENABLE=true
export MEMORY_LIMIT=512
export CPU_MAX_OCCUPY=95
./agent-leak-app &
# 60초간 워커 로그 누적
sleep 60
grep -E "Worker-[A-C]" $AGENT_LOG_DIR/agent-leak-app.log > evidence/scheduling_workers.log
top -H -bn1 -p $(pgrep -f agent-leak-app | head -n1) > evidence/scheduling_top_h.txt
```

→ 분석: [reports/04_scheduling_analysis.md](reports/04_scheduling_analysis.md)
